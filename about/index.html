关于文章的题目

我个人认为一个人对事物理解的深入程度分为 3 个层次，以技术为例：

  • [What] -> 这个东西是什么，如何使用它？

    最基本的从应用层的层面来用熟练一个东西，快速入门

  • [How] -> 它内部具体是如何实现的，我可以仿照这种实现在我的哪些应用中？

    通过分析内部原理来优化自己的一些应用（比如参考 linux 的初始化列表，我们也可以仿照其来编辑链接脚本生成我们自己的初始化列表）

  • [Why] -> 它内部为什么这么实现，为什么不那么实现，我是否有更好的方式可以将它优化？

    领会作者的意图，并在此基础上优化。（比如给作者发送 Pull Request）
