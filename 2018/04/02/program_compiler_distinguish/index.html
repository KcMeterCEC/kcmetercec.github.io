[What]compiler -> 区分编译器及系统

发表于 2018-04-02 更新于 2020-08-01 分类于 program ， compiler 阅读次数： Valine：

关于编译器和系统的区分老是忘记，故在此记录一下。

在区分编译器和系统时，都是使用下面这个方式来判断对应宏：

区分编译器

gcc

gcc具有以下宏来标识自己： 

__GNUC__
__GNUC_MINOR__
__GNUC_PATCHLEVEL__

实际上这些宏都是带有数字的，以表示gcc的版本号。

  • 如果仅仅是判断一个编译器是不是gcc，那么使用以下方式即可
  • 如果对应的代码需要对于gcc不同版本,需要用以下方式

Visual C++

visual c++ 下判断最常用的就是: 

_MSC_FULL_VER
_MSC_VER
  • 如果仅仅是判断一个编译器是不是 visual c++，那么使用以下方式即可
  • 如果要判断版本则需要以下方式

区分系统

通过CPU字长区分

在嵌入式开发中，应用代码经常在PC上进行模拟，模拟无误后再在目标板上执行。 现在PC基本上都是64位了，所以可以通过其字长来判断当前代码是在PC上运行还是在目标机上运行。

通过系统类型区分

不同的系统定义了不同的宏：

  • windows
    • _WIN32
    • _WIN64
  • linux
    • __linux__
